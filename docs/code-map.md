# Code map

The repository currently has 193 top-level Python files because it preserves
the code that produced and reviewed many frozen experiments. That is provenance,
not the supported public API.

## Supported implementation surface

These modules contain the reusable process and measurement logic:

| Module | Responsibility |
|---|---|
| `tsv_process.py` | Starting geometry and process-step helpers |
| `process_config.py` | Typed access to `config/process.toml` |
| `traveler_metrics.py` | Traveler geometry, topology, and connectivity |
| `profile_shape_metrics.py` | Wall and floor shape diagnostics |
| `full_2d_layer_metrics.py` | Full-width regional film measurements |
| `layer_process_models.py` | Liner, barrier, and seed models |
| `morphology_fill_control.py` | Copper morphology controls |
| `copper_fill_transport_3d.py` | 3D transport bridge |
| `native_domain_checkpoint.py` | Native ViennaPS save/load boundary |
| `legacy_metric_guard.py` | Prevents retired metrics from re-entering the active path |

The public entry points are `explainer.html`, `CASE_STUDY.md`, and `README.md`.
The static guide is generated by `build_explainer.py` from committed publication
data.

## File categories

Snapshot on 19 July 2026:

| Category | Count | Rule |
|---|---:|---|
| Public core | 25 | the ten modules above and their focused tests |
| Publication builders and checks | 33 | guide/data builders, schemas, and publication tests |
| Active research and evidence production | 134 | campaign builders, runners, reviews, freezers, and their tests |
| Likely dead | 1 | removed orphan interim plotting script |

Naming conventions provide the complete root classification:

- `test_*.py`: executable regression or evidence check;
- `build_*.py`: immutable design, evidence, or publication builder;
- `foundation_*.py`, `*_runner.py`, `run_*.py`, and `watch_*.py`: experiment
  execution;
- `review_*.py`: evidence reconciliation and bounded scientific decision;
- `freeze_*.py`: manifest/version guard;
- the ten modules above: reusable implementation.

`scripts/run_capability_tests.py` is the authoritative runtime routing inventory
for tests. Files under `archive/` are historical and have no current claim
authority.

## Why the research files have not been mass-moved

Frozen manifests and reviews record source paths and hashes. Moving 134 campaign
files in one cleanup commit would make the root prettier while weakening those
citations and creating a large import rewrite. The safe migration is versioned:

1. publish an exhaustive evidence index;
2. copy required executed sources into their study version;
3. update active imports and test routing;
4. verify old hashes remain resolvable; and
5. only then move active research into `research/` and tests into `tests/`.

Until that migration is complete, clarity comes from this small supported
surface, not from pretending every root script is a library API.

## Intended later layout

```text
src/viennaps_hbm/   reusable process, model, checkpoint, and metric code
publication/        guide builders and release projections
research/           active Bosch, copper, CMP, and numerical campaigns
tests/              core, publication, and research checks
evidence/           immutable study evidence and reviews
docs/               method, status, and public case study support
archive/            withdrawn or superseded work
```

This is a migration target, not a request to add a packaging framework now.
