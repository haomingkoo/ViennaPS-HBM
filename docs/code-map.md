# Code map

The repository preserves the code that produced and reviewed frozen experiments.
That code is provenance, not the supported public API. To inspect the current
root inventory without relying on a copied count, run:

```bash
git ls-files | awk 'index($0, "/") == 0'
```

## Supported implementation surface

These modules contain the reusable process and measurement logic:

| Module | Responsibility |
|---|---|
| `tsv_process.py` | Starting geometry and process-step helpers |
| `process_config.py` | Typed access to `config/process.toml` |
| `traveler_metrics.py` | Traveler geometry, topology, and connectivity |
| `profile_shape_metrics.py` | Wall and floor shape diagnostics |
| `full_2d_layer_metrics.py` | Full-width regional film measurements |
| `layer_process_models.py` | Liner, barrier, and seed models |
| `morphology_fill_control.py` | Copper morphology controls |
| `copper_fill_transport_3d.py` | 3D transport bridge |
| `native_domain_checkpoint.py` | Native ViennaPS save/load boundary |
| `legacy_metric_guard.py` | Prevents retired metrics from re-entering the active path |

The public entry points are `explainer.html`, `CASE_STUDY.md`, and `README.md`.
The static guide is generated by `scripts/build/build_explainer.py` from
committed publication data.

## File categories

| Category | Rule |
|---|---|
| Public core | The reusable modules above and their focused tests |
| Publication | Guide and data builders, schemas, and publication tests |
| Active studies | Campaign designs, runners, reviews, freezers, and their tests |
| Historical work | Superseded campaigns indexed under `archive/` |

Naming conventions provide the remaining root classification:

- `tests/test_*.py`: executable regression or evidence check;
- `scripts/build/build_*.py`: active publication or evidence builder;
- root `build_*.py`: path- and hash-frozen campaign builder;
- `foundation_*.py`, `*_runner.py`, `run_*.py`, and `watch_*.py`: experiment
  execution;
- `review_*.py`: evidence reconciliation and bounded scientific decision;
- `freeze_*.py`: manifest/version guard;
- the ten modules above: reusable implementation.

`scripts/run_capability_tests.py` is the authoritative runtime routing inventory
for tests. Files under `archive/` are historical and have no current claim
authority.

## Why some research files remain at the root

Frozen manifests and reviews record source paths and hashes. Tests and
provenance-safe builders have moved to `tests/` and `scripts/build/`.
Twenty-three executed or dependency-bound builders remain at their root paths.
Moving those files requires:

1. publish an exhaustive evidence index;
2. copy required executed sources into their study version;
3. update active imports and test routing;
4. verify old hashes remain resolvable; and
5. only then move active research into `research/` and tests into `tests/`.

Until that archive exists, the cited builders stay unchanged. Root compatibility
wrappers are intentionally avoided because they would preserve the clutter
without preserving the executed bytes.

## Current layout

```text
tests/          verification
evidence/       immutable results and provenance
scripts/build/  active publication and evidence builders
docs/           method, status, and public guidance
scripts/        repository build and CI utilities only
archive/        withdrawn or superseded work
root *.py       reusable modules and provenance-bound campaign code
```

Reusable process modules remain at the root for now because the project is not
an installable Python package. A `src/` layer should be added only with a real
`pyproject.toml` and editable-install workflow.
