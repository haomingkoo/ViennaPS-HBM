# Domain Docs

This repo uses a single domain context.

## Before exploring, read these

- `CONTEXT.md` at the repo root
- Relevant ADRs under `docs/adr/`
- `program.md`, `train.md`, and `prepare.md`, in that order

## Use the glossary's vocabulary

When outputs name a TSV process concept, use the term as defined in `CONTEXT.md`. If a needed term is missing, add it only when the work resolves the meaning.

## Flag ADR conflicts

If a proposal contradicts an ADR, name the ADR and explain why reopening it is justified.
