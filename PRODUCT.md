# Product

## Register

brand

## Users

Engineers learning ViennaPS, technical hiring panels, semiconductor process
engineers, and simulation researchers. A new reader should understand the repo
and run a canonical traveler; a deeper reader should be able to audit what was
tested, what failed, why the conclusions are credible, and where the model
stops representing physical processing.

## Product Purpose

Teach the ViennaPS-HBM repository through an auditable, interactive TSV case
study. The report first explains the package, model units, repo map, and how to
run a traveler. It then shows how to use the same machinery for screening,
controlled interactions, replication, boundary expansion, and process-window
analysis. A reader should be able to trace each conclusion from target
specification through hypothesis, experiment, visual geometry, and the final
recommendation or model limitation.

## Brand Personality

Measured, forensic, candid. The report should feel like an instrument readout
annotated by a careful researcher, with enough warmth and narrative rhythm to
remain readable as a long-form case study.

## Anti-references

No startup landing-page treatment, glossy success narrative, decorative
scientific imagery, unexplained scalar scoreboards, or claims that turn model
limitations into physical-process conclusions. Avoid repetitive cards,
generic dashboards, and interactions that interpolate or invent data.

## Design Principles

1. Put the decision and its hard limits before the history.
2. Give first-time users an overview and a reproducible path through the repo.
3. Let readers inspect sampled evidence, seeds, gates, and geometries.
4. Keep failed hypotheses and retractions visible.
5. Separate observation, interpretation, and recommendation.
6. Use the same gate-first ranking logic as the research campaign.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-operable controls, meaningful focus states,
non-color status labels, responsive tables and figures, and reduced-motion
support. Every visual conclusion must also be stated in text.
